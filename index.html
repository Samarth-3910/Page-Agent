Tell this page what to do.

Page Agent runs as plain JavaScript inside a web page, reads the live DOM as text, and turns a sentence into clicks, keystrokes, and form fills. Try it below.

! Before you try this: the free demo below runs on Page Agent's public demo LLM, which routes through Alibaba's infrastructure in China. Whatever you type into the command box and the page content it reads to act on your command is sent to that service. Don't enter real names, emails, passwords, or any other sensitive or personal information here; use the sample data already in the fields or made-up values instead. Read the full documentation and terms before using this in anything beyond casual testing.

Get started

Add it in one line, or wire it up properly.

CDN — one line
npm — production
<!-- Evaluation only — uses Page Agent's free demo LLM -->
<script src="https://cdn.jsdelivr.net/npm/page-agent@1.10.0/dist/iife/page-agent.demo.js" crossorigin="true"></script>
import { PageAgent } from 'page-agent'

const agent = new PageAgent({
  model: 'qwen3.5-plus',
  baseURL: 'https://dashscope.aliyuncs.com/compatible-mode/v1',
  apiKey: 'YOUR_API_KEY',
  language: 'en-US',
})

await agent.execute('Click the login button')
How it works

Three steps, no server round-trip.

Everything happens inside the page the visitor is already looking at — so the agent inherits their cookies, session, and auth for free.

01 / read

Dehydrate the DOM

On each command, Page Agent scans the live document, pulls out every interactive element — buttons, links, inputs — and compresses it into compact text.

FlatDomTree
02 / reason

Ask any LLM

That text goes to a model you choose over any OpenAI-compatible endpoint — Qwen, GPT, Claude, or a local Ollama model. No pixels, no vision costs.

model-agnostic
03 / act

Execute in place

The plan comes back as a sequence of real DOM actions — click, type, select — dispatched as native browser events, right where the user already is.

native events
Where it fits

An agent that lives with the user, not outside the browser.

Playwright and Selenium drive a separate browser process from the outside. Page Agent runs where the user already is.

Approach Runs Reads the page via Needs a model that sees images Inherits the user's session

Playwright / Selenium External process Accessibility tree / CDP No No
